# Seat broker

Hands out pre-provisioned seats (namespace + scoped kubeconfig + URL) to attendees by name,
idempotently, and serves a live progress board for the projector. Stdlib-only Go.

## Run

```sh
SEATS_FILE=seats.json ADDR=:8088 go run main.go
# board:   http://localhost:8088/
# claim:   curl -XPOST localhost:8088/claim -d '{"name":"dan"}'
# status:  curl localhost:8088/status
```

## seats.json

An array of seats — see `seats.example.json`. Generated by the cluster bootstrap (Phase 4):
for each `ws-aNN` namespace, read the `attendee-token` Secret + cluster CA/endpoint and emit a
kubeconfig whose **only** credential is that namespace-scoped ServiceAccount token (no GCP IAM).
`claims.json` is written next to the broker and survives restarts.

## Endpoints

| Method | Path | Body | Returns |
|---|---|---|---|
| POST | `/claim` | `{"name":"dan"}` | the assigned `Seat` (same seat on repeat) |
| POST | `/progress` | `{"id":"a01","step":"steal"}` | 204 — feeds the board |
| GET | `/status` | — | `{total, claimed, seats:[{ID,ClaimedBy,Step}]}` |
| GET | `/` | — | HTML progress board |

## Deploy

Single static binary (`CGO_ENABLED=0 go build`). Run anywhere attendees can reach — a tiny VM,
Cloud Run, or even a laptop on the same network. Front it with TLS if it's public.
